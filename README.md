# Система AI-Контроля Качества (AI Quality Inspector)

## Описание
Данный проект представляет собой систему автоматизированной детекции биологических дефектов (таких как сгустки крови, кровоподтеки, фрагменты костей) на поверхности говядины с использованием VLM (Vision Language Models). Система анализирует визуальный ряд в реальном времени, выявляя аномалии на основе эталонных изображений и отфильтровывая фоновый шум (элементы оборудования, спецодежду рабочих, ленты конвейера).

**Статус проекта:** Проект является демонстрационным (Proof-of-Concept).

## Источник данных (RTSP Камеры)
На данный момент система работает с предзаписанным видеофайлом `production_with_realistic_defects.mp4` для демонстрации своих аналитических возможностей. 

Система уже сейчас **полностью поддерживает работу с IP-камерами по протоколу RTSP**.
Для перехода с тестового видеофайла на реальную промышленную камеру, достаточно изменить аргумент `src` при инициализации класса `CameraStream` в файле `main.py` на RTSP-адрес вашей камеры:
```python
# Было:
camera = CameraStream(src='production_with_realistic_defects.mp4').start()

# Стало (пример):
camera = CameraStream(src='rtsp://admin:password@192.168.1.100:554/stream1').start()
```

## Демонстрация
Основное демо работы системы записано в файле `presentation_video.mp4`. Ознакомьтесь с ним, чтобы увидеть, как система обрабатывает реальный видеопоток, накладывая ограничительные рамки (bounding boxes) на обнаруженные дефекты.

## Стек и Технические Особенности
- **Технологии:** Python (3.10+), OpenCV (работа с видеопотоком и UI), мультимодальные модели (VLM / LMM).
- **Приватность и Безопасность (Local AI):** Использование локального сервера **LM Studio** позволяет обрабатывать данные с производственных камер **без отправки конфиденциального видео в облако** (как это было бы с OpenAI или Anthropic). Это критично для сохранения коммерческой тайны и соблюдения стандартов безопасности пищевого производства.
- **Инфраструктура:** Проект работает через **LM Studio** (локальный сервер), а не через Ollama. Приложение обращается к серверу, использующему OpenAI API совместимый эндпоинт.
- **Используемая модель:** Для максимальной производительности и понимания предметной области настоятельно рекомендуется `qwen/qwen3-vl-8b` (в качестве альтернативы могут применяться её аналоги из семейств Gemma или Llava).

## Инструкция по настройке и запуску

### 1. Установка зависимостей
Склонируйте конфигурацию проекта локально и установите необходимые Python библиотеки:
```bash
pip install -r requirements.txt
```

### 2. Подробная настройка LM Studio
* Загрузите приложение [LM Studio](https://lmstudio.ai/).
* Скачайте модель из семейства Qwen (например, Qwen-VL-Chat или Qwen3-VL) или аналогичную VLM.
* Перейдите во вкладку **Local Server**.
* В настройках сервера задайте следующие параметры:
  *   **Port:** `1234`
  *   **Context Length:** `13676` (необходимо для корректной загрузки и обработки как изображений датасета, так и текущего кадра).
  *   **Vision Adapter:** Обязательная активация (без этого сервер не сможет интерпретировать передаваемые кадры в Base64).
* Нажмите кнопку **Start Server**.

### 3. Запуск системы и Управление
Запустите основной скрипт анализа:
```bash
python main.py
```

Управление видеопотоком осуществляется с помощью следующих "горячих клавиш" на клавиатуре:
*   `s` — **save standard:** сохранить текущий чистый кадр как эталон "идеальной продукции" (Positive Reference).
*   `d` — **save defect:** сохранить текущий кадр как обнаруженный системный дефект (Negative Reference). *Примечание: в реальных условиях потребуется обвести дефект на фото маркером для точного RAG целеуказания.*
*   `q` — завершить работу программы.

## Ограничения и Масштабируемость
Текущая медленная скорость инференса и обновления счетчиков обусловлена исключительно ограничениями тестового (домашнего) железа. 

В условиях реального цеха, использование промышленных GPU решений (например, серверных плат NVIDIA A100 или мощнее) позволит:
* Запускать значительно более массивные ("тяжелые") модели с высоким контекстом внимания.
* Выполнять их полноценное дообучение (**Full Fine-tuning**) на тысячах спецификаций дефектов.
* Добиться практически мгновенной классификации продукции в режиме реального времени даже на очень высоких скоростях конвейерной ленты.
